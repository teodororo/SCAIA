GitHub Action · Marketplace

Revisão de código
por IA em cada PR

Uma GitHub Action reutilizável que lê os diffs do seu pull request e posta comentários inline com os achados. Traga seu próprio token — funciona com qualquer endpoint OpenAI-compatível.

Fluxo

Como funciona

01

Leitura do diff

No evento pull_request, a action lista os arquivos alterados e extrai seus diffs, respeitando o limite configurado em max-files.

02

Anotação de linhas

Cada diff é anotado com os números de linha do lado novo (hunks do git) antes de ser enviado ao modelo, permitindo comentários inline precisos.

03

Chamada ao modelo

O diff anotado é enviado ao endpoint configurado — OpenAI, OpenRouter, Claude via proxy, ou qualquer modelo compatível com a API Chat Completions.

04

Achados como comentários

O modelo retorna JSON estruturado (caminho, linha, severidade, comentário). Cada achado vira um comentário inline no PR; os fora do alcance do diff são incorporados ao resumo.

Integração

Adicione em minutos

.github/workflows/ai-review.yml 
name: Revisão de Código por IA
on:
  pull_request:
permissions:
  contents: read
  pull-requests: write
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: teodororo/SCAIA@v1
        with:
          api-token: ${{ secrets.AI_API_TOKEN }}
          model: gpt-4o
.github/workflows/ai-review.yml 
- uses: teodororo/SCAIA@v1
  with:
    api-token: ${{ secrets.OPENROUTER_KEY }}
    api-base-url: https://openrouter.ai/api/v1
    model: anthropic/claude-3.5-sonnet
.github/workflows/ai-review.yml 
- uses: teodororo/SCAIA@v1
  with:
    api-token: ${{ secrets.MY_AI_KEY }}
    api-base-url: https://meu-proxy.exemplo.com/v1
    model: meu-modelo-local
        # Qualquer endpoint OpenAI-compatível funciona

Referência

Inputs e outputs

Input Padrão Descrição
api-token obrigatório — Chave de API para o provedor de IA escolhido.
model obrigatório — ID do modelo a usar (ex: gpt-4o).
api-base-url opcional openai.com/v1 URL base do endpoint Chat Completions.
github-token opcional github.token Token para ler o PR e postar comentários.
system-prompt opcional embutido Override do prompt de sistema do revisor.
max-files opcional 50 Limite de arquivos alterados enviados à IA.
exclude opcional — Globs separados por vírgula para ignorar arquivos.
fail-on-findings opcional false Falha o job quando qualquer achado é reportado.

Output: findings-count — número de achados produzidos pela IA.

Destaques

Traga seu próprio modelo

🔑

Seu token, seu custo

Nenhum segredo passa por servidores de terceiros. A chave de API vai direto do seu repositório para o provedor.

🔌

Qualquer provider

OpenAI, OpenRouter, Anthropic via proxy, Mistral, Groq ou modelo self-hosted — qualquer endpoint /v1/chat/completions.

💬

Comentários inline

Os achados vão para as linhas exatas do diff. O que fica fora do alcance é incluído no resumo da revisão.
